## 一、 五個排序法介紹
1.氣泡排序法:
氣泡排序透過重複走訪數列，比較相鄰元素並在順序錯誤時交換，使較大值逐步「浮」向末端。

2.選擇排序法:
選擇排序每次從未排序部分尋找最小值，並將其與起始位置交換。

3.插入排序法:
插入排序將數列分為已排序與未排序兩部分，逐一將未排序元素插入已排序序列的正確位置。

4.合併排序法:
合併排序利用分治法將數列不斷切半直至單一元素，再進行有序合併，效能極為穩定。

5.快速排序法:
快速排序同樣採用分治法，選取基準值將數列分為兩部分再遞迴排序，是實務中最常用的演算法。

## 二、 複雜度分析
| 排序法 | 平均時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- |
| 冒泡排序 | $O(n^2)$ | $O(1)$ | 穩定 |
| 選擇排序 | $O(n^2)$ | $O(1)$ | 不穩定 |
| 插入排序 | $O(n^2)$ | $O(1)$ | 穩定 |
| 快速排序 | $O(n \log n)$ | $O(\log n)$ | 不穩定 |
| 合併排序 | $O(n \log n)$ | $O(n)$ | 穩定 |

## 四、 結果比較與個人心得
從實驗數據可以明顯觀察到，當數據量 $n$ 增加時，**冒泡排序**的執行時間增加最為劇烈，在 $n=5000$ 時已接近 0.7 秒；相對地，**快速排序**與**合併排序**即使在數據量增加時，執行時間仍維持在 0.01 秒以下，展現了 $O(n \log n)$ 演算法在處理大規模數據時的壓倒性優勢。
