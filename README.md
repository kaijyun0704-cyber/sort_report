## 一、 五個排序法介紹
**1.氣泡排序法:**

氣泡排序是一種基礎的交換排序方式，其運作核心在於重複走訪待排序的數列，每次比較相鄰的兩個元素，若發現前者的數值大於後者則進行交換，使較大的數值如同氣泡般逐步往數列末端移動。這個過程會持續重複直到整份數列完全不需要再進行任何交換為止。

**2.選擇排序法:**

選擇排序的操作邏輯非常直觀，它將數列分為已排序與未排序兩部分，每次從未排序的序列中尋找最小值，並將其與未排序部分的起始位置進行交換。這種方式不論原始數據的排列狀態如何，每一輪都必須掃描剩下的所有元素以確定最小值，雖然效率較低，但其交換次數相對較少。

**3.插入排序法:**

插入排序的原理類似於整理撲克牌的手法，其運作方式是將第一個元素視為已排序，隨後從第二個元素開始逐一取出，並由後向前掃描已排序部分的序列，找到合適的位置後將元素插入。這種演算法在數據量較小或是數列已經接近排序完成的情況下，表現非常優異。

**4.合併排序法:**

快速排序是基於分治法發展出的高效演算法，它會從數列中選取一個基準點，並將數列重新排列，使得所有比基準點小的數值移至左側，比基準點大的數值移至右側，最後再透過遞迴的方式對左右兩個子數列進行同樣的處理。由於其處理速度極快，是目前實務應用中普及率最高的排序法之一。

**5.快速排序法:**

合併排序同樣採用分治法，但其策略是先將數列不斷對半切割，直到每個子序列都只剩下一個元素，接著再將這些子序列兩兩合併成一個有序的數列。這種方法的優點在於效能極其穩定，且具備穩定排序的特性，不會改變相同數值元素的原始相對順序。

## 二、 複雜度分析
| 排序法 | 平均時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- |
| 氣泡排序 | $O(n^2)$ | $O(1)$ | 穩定 |
| 選擇排序 | $O(n^2)$ | $O(1)$ | 不穩定 |
| 插入排序 | $O(n^2)$ | $O(1)$ | 穩定 |
| 快速排序 | $O(n \log n)$ | $O(\log n)$ | 不穩定 |
| 合併排序 | $O(n \log n)$ | $O(n)$ | 穩定 |

## 四、 結果呈現與比較
以下為在本人環境下執行 `sorting.py` 所得之真實測試結果（單位：秒）：

| 數據規模 (n) | 氣泡排序 | 選擇排序 | 插入排序 | 快速排序 | 合併排序 |

| 5,000 | 0.9937 | 0.4196 | 0.4019 | 0.0079 | 0.0094 |
### 實驗觀察
根據實驗數據可以觀察到，當數據規模 $n$ 從1,000增加至5,000時，氣泡排序、選擇排序與插入排序的執行時間呈現大幅度的指數增長。其中氣泡排序在 $n=5,000$ 時耗時接近 1 秒，為所有演算法中最慢的一種，這印證了 $O(n^2)$ 演算法在處理大量數據時的侷限性。雖然選擇排序與插入排序的表現略優於氣泡排序，但其執行時間仍隨著數據量增加而顯著上升。相對而言，快速排序與合併排序展現了極高的效能優勢。在 $n=5,000$ 的規模下，快速排序僅需約0.0079秒即可完成，與氣泡排序相比有超過百倍的效能差距。這說明了 $O(n \log n)$ 類型的演算法在處理大規模數據時具備絕對的優勢，且快速排序在實測中通常比合併排序更為迅速，符合其作為實務主流排序法的特性。

## 五、 心得與結論
透過本次針對五種排序演算法的實作與數據分析，我深刻體會到演算法理論中「時間複雜度」在現實執行環境中的具體影響。在實驗初期，當處理的數據量較小時，各種排序法之間的耗時差距極其微小，甚至在毫秒級別下難以察覺。然而，隨著數據規模 $n$ 的提升，演算法優劣的影響便會被顯著放大。從實測數據中可以清楚看到， $O(n^2)$ 等級的氣泡排序、選擇排序與插入排序，在面對大量資料時會產生嚴重的執行延遲，而 $O(n \log n)$ 的快速排序與合併排序則展現了極佳的處理效率與擴充性。在個人觀察方面，我也注意到了演算法在不同情境下的特性差異。例如插入排序雖然在大規模數據下表現不佳，但在數據接近排序完成的情況下，其執行速度反而非常驚人，這證明了沒有絕對完美的演算法，只有最適合當前場景的選擇。而在實作計時功能的過程中，我也學習到如何將抽象的邏輯轉化為具體的數據，並透過量化分析來驗證理論的正確性。這種從「寫程式」到「分析程式效能」的轉變，是我在這次作業中收穫最大的部分。總結來說，這次作業不僅讓我掌握了基礎的排序邏輯，更讓我了解到在開發實務中，根據資料規模選擇合適演算法的重要性。雖然快速排序在大多數測試中效能最優，但合併排序的穩定性在特定場景下也具備不可替代的價值。這次的實驗分析讓我對程式優化與資料處理有了更直觀且具體的認識，也讓我學會了如何撰寫一份兼具理論基礎與實驗證據的完整分析報告。
